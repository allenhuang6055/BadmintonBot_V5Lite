# 健好羽球 LINE Bot V9.2.2 穩定修正版

## 本版只修現有功能錯誤，不新增功能

修正內容：

1. 交款優先判斷，避免被收入模糊辨識誤抓。
2. 支出第二優先，避免被收入「其他」誤抓。
3. 收入最後判斷。
4. `expense.js` 不再從 `income.js` 匯入解析器，避免收入修改影響支出。
5. `payment.js` 不再從 `income.js` 匯入解析器，避免收入修改影響交款。
6. 支出也支援基本容錯，例如 `餐點4` 會辨識為 `聚餐4`。
7. 查詢指令 `今天`、`庫存`、`月報` 不會寫入資料庫。

## 必測案例

### 收入

```text
零打：1200元
球劵：1600元
贊助：1000元
耗球：20顆
```

應為收入合計 3,800，支出/交款不得觸發。

### 支出

```text
支出
買球5
場租66
餐點4
雜支4
其他4
```

應走支出流程，不得走收入。

### 交款

```text
幹部交款
交款：4000000
備註：
```

應走交款流程，不得走收入或支出。

### 查詢

```text
今天
庫存
月報
```

只回覆查詢，不得寫入任何資料。

## Render Logs

會看到：

```text
PARSE_INCOME_RESULT
PARSE_EXPENSE_RESULT
PARSE_PAYMENT_RESULT
```

用來確認解析結果。

補充修正：明確屬於交款的文字不會再被收入的贊助/捐款模糊辨識誤抓。
