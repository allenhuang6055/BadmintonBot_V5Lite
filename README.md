# 健好羽球 LINE Bot V10 狀態流程穩定版

## 本版核心修正

改成 Session 狀態流程。

也就是：

```text
輸入「收入」
↓
下一則訊息一定走收入
```

```text
輸入「支出」
↓
下一則訊息一定走支出
```

```text
輸入「交款」
↓
下一則訊息一定走交款
```

這樣可以完全避免：

- 收入被判成支出
- 支出被判成收入
- 交款被判成贊助
- 因為「其他」同時存在於收入/支出導致誤判

## 使用方式

### 收入

```text
收入
```

接著輸入：

```text
零打：500
球券：600
會員：800
其他：890
耗球：59
備註：
```

### 支出

```text
支出
```

接著輸入：

```text
買球：5
場租：66
餐點：4
雜支：4
其他：4
備註：
```

### 交款

```text
交款
```

接著輸入：

```text
交款：4000000
備註：
```

## 查詢

以下指令只查詢，不會寫入資料庫：

```text
今天
庫存
月報
我的未交
```

## 取消操作

如果輸入錯模式，可輸入：

```text
取消
```

## 本版也移除辨識項目提示

LINE 畫面不再顯示「已將 XXX 辨識為 XXX」或一堆辨識項目。
解析細節保留在 Render Logs。

# V10.1 更新：補單日期與庫存檢查

## 補單日期

收入、支出、交款都支援補日期。

範例：

```text
收入
日期：2026/07/01
零打：500
球券：600
耗球：59
備註：補單
```

也可以：

```text
支出
日期：7/1
場租：1000
備註：補單
```

沒有輸入日期時，一律使用今天。

## 羽球庫存檢查

耗球會寫入 `02_LINE資料庫` 的「最終耗球」欄位；`庫存` 查詢會重新從資料庫加總。

如果庫存沒有變，請檢查：

1. `02_LINE資料庫` 是否有一列 `類型=庫存 / 項目=耗球`
2. 該列 `Y欄 最終耗球` 是否有數字
3. `L欄 狀態` 是否為 `有效`
4. `00_首頁!B6` 是否有填期初庫存顆數
